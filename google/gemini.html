拿到 Gemini API Key

n8n 裝好了，現在要拿一把「AI 鑰匙」帶回去給它用。這把鑰匙叫做 API Key，用來呼叫 Google 的 Gemini AI 模型。我們走老師上課教的 GCP Console 路線。

Google Cloud Console 小白友善 免費額度夠用

整個流程長怎樣

分三段。第一段先進 Google Cloud（PART 1，三步）。第二段開啟 Gemini API 功能（PART 2，三步）。第三段拿到那把 API Key 並收好（PART 3，三步）。整體大約 15 分鐘。

等一下，API Key 是什麼？為什麼要從 GCP 拿？

API Key 就是「給程式用的密碼」。Gemini AI 不是給你隨便用的，要先拿一把屬於你的鑰匙，n8n 拿著這把鑰匙才能呼叫 Gemini 服務。

GCP（Google Cloud Platform）是 Google 的雲端服務後台，所有 Google API（Gemini、Sheets、Gmail 之類）的鑰匙都在這裡管。

為什麼不直接用 AI Studio？AI Studio 是另一條更快的捷徑，但老師選教 GCP Console 是有原因的：(1) GCP Console 是更通用的方式，未來想拿 Sheets 或 Gmail 的鑰匙都用同一個地方。(2) 學會 GCP Console 等於學會 Google 整個雲端世界的入口。

需要綁信用卡嗎？會要你綁。但 Gemini 有很大的免費額度（每天 1500 次請求免費），自己練習一年用不完。除非你蓋商業服務，不然不會跳出來收錢。

PART 1
進入 Google Cloud
登入 GCP Console、同意條款、開啟計費。這段是地基，所有 Google API 都要過這關。
一

登入 Google Cloud Console

為什麼要做這件事

GCP Console 是 Google 雲端服務的後台，所有 API 的開關跟鑰匙都在這裡。要拿 Gemini Key 之前，要先進來這個後台。

第一次進去，Google 會要你做一些初始設定。後面的人有 Google 帳號就能直接用。

操作步驟
  1. 打開瀏覽器，去 console.cloud.google.com。
  2. 用你常用的 Google 帳號登入（就是 Gmail 那個帳號）。建議用你自己的個人帳號，不要用學校或公司給的，未來比較好管。
  3. 進去後會看到一個藍色頂端的後台畫面。看到了就成功一半。
前往 Google Cloud Console
二

同意條款 + 設定國家

為什麼要做這件事

第一次用 GCP，Google 會跳一個視窗要你同意服務條款，並選你的國家（影響稅金跟付款幣別）。沒同意就不能用。

這跟你裝任何新軟體第一次跳出的「使用者授權合約」差不多，照做就好。

操作步驟
  1. 跳出視窗後，Country 選擇 Taiwan（如果你在台灣）。
  2. 勾選下面那個「我同意服務條款」（Terms of Service）。
  3. 下面那個「想收 Google 行銷信」的勾要不要勾隨便你。
  4. 按 Agree and Continue（同意並繼續）。
三

開啟計費（綁信用卡，但放心）

為什麼要做這件事

GCP 要求所有人都要綁一張信用卡才能用，就算你只想用免費的功能也要綁。這是 Google 防止有人開無數個帳號免費用的機制。

但放心，Gemini API 給的免費額度很大（每天 1500 次請求免費）。除非你蓋大型商業服務，不然根本用不完，也不會跳出來扣錢。

有一個保險：第一次綁卡可以拿 300 美金免費試用額度，90 天內用不完就過期，期間 GCP 也不會自動扣你錢。

操作步驟
  1. 同意條款後，畫面會引導你到「啟用免費試用」（Start Free Trial）的頁面。
  2. 填寫個人資料、地址、信用卡資訊。用真實資料，假資料會被擋。
  3. 按 開始我的免費試用。
  4. 完成後就會回到 GCP Console 主畫面，這時候你已經是正式用戶了。
怕自動扣款怎麼辦？之後可以在 Billing 頁面設「預算與警示」，每月超過某金額就寄信通知你。對小白來說最簡單做法：設 0 元預算 + 警示，一花到錢馬上就收到信。
PART 2
啟用 Gemini API 功能
GCP 預設什麼 API 都沒開，要去 API Library 裡把 Gemini 那個開關打開。
一

建一個新的 Project

為什麼要做這件事

GCP 用 Project（專案） 來管理所有資源。一個 Project 就像一個資料夾，裡面放著你用到的所有 API、Key、計費紀錄。

第一次用會自動有一個叫 My First Project 的預設 Project，可以直接用。但為了清楚，我們建一個專門給 n8n 用的，叫 n8n-gemini。

操作步驟
  1. 在 Console 頂端中央，點 Project 名稱的下拉選單（顯示 My First Project 那一塊）。
  2. 跳出視窗後，點右上角 NEW PROJECT（新專案）。
  3. Project Name 填 n8n-gemini（或你喜歡的名字）。
  4. 按 CREATE，等個 5-10 秒，Project 就建好了。
  5. 再回頂端下拉選單，選剛建好的 n8n-gemini，確認頂端顯示這個名字。
二

搜尋「Generative Language API」

為什麼要做這件事

GCP 裡有上百個 API，預設全部關著。要用 Gemini，要先去找它然後打開開關。

注意：Gemini API 在 GCP 裡的正式名稱叫 Generative Language API（生成式語言 API）。搜尋的時候要用這個名字，搜 gemini 不一定找得到。

操作步驟
  1. 在 Console 頂端的搜尋列，輸入 Generative Language API。
  2. 會跳出搜尋結果，找到 Generative Language API（藍色標題、有 Google 圖示），點下去。
  3. 會進到那個 API 的介紹頁面。
三

按 ENABLE 啟用

為什麼要做這件事

進入 API 介紹頁後，預設是「未啟用」狀態。按一下 ENABLE（啟用）按鈕，這個 API 才會真正開放給你用。

啟用只要按一次，之後永遠是開的，除非你手動關掉。

操作步驟
  1. 在 Generative Language API 頁面，看畫面中央有一顆藍色 ENABLE 按鈕。
  2. 按下去，等 5-15 秒，畫面會跳到「已啟用」（API enabled）狀態。
  3. 看到綠色勾勾或 API enabled 字樣，就代表開好了。
PART 3
拿到你的 API Key
這段最關鍵。Key 拿到後就能去 n8n 用，但拿了之後一定要收好。
一

進「Credentials」頁面

為什麼要做這件事

Credentials（憑證）就是 GCP 管理所有 Key 的中央倉庫。你拿到的 API Key 會存在這裡，之後想刪、想換新都從這裡操作。

操作步驟
  1. 在 Console 左上角，點 選單圖示（三條橫線）。
  2. 找 APIs & Services，把滑鼠移上去，會展開子選單。
  3. 子選單裡點 Credentials（憑證）。
  4. 進到 Credentials 頁面，預設是空的（沒有任何 Key）。
二

建立 API Key

為什麼要做這件事

到了 Credentials 頁面，按一個按鈕就能產出新的 API Key。Google 會幫你產一串又長又亂的字元（例如 AIzaSyD_xxxxxxx），這就是你的鑰匙。

每次按都會產一支新的，你可以同時有很多支 Key。建議一個用途一支，方便管理。

操作步驟
  1. 在 Credentials 頁面頂端，按 + CREATE CREDENTIALS（建立憑證）。
  2. 下拉選單中選 API key。
  3. 跳出視窗，會顯示你新產出的 Key，是一串以 AIzaSy 開頭的字串。
三

複製 Key，貼到安全的地方

為什麼要做這件事

剛跳出的視窗是你看到完整 Key 的最後機會。關掉之後，Credentials 列表只會顯示 Key 的前幾個字（為了安全）。

所以這時候要立刻把 Key 複製起來，貼到一個安全的地方。建議用密碼管理器（1Password、Bitwarden 等），或一個你不會推到 GitHub 的本機檔案。

操作步驟
  1. 視窗裡 Key 的右邊有一個 複製圖示，點下去把 Key 複製到剪貼簿。
  2. 貼到密碼管理器、Notes、或 secrets.txt 之類的安全位置。千萬不要貼到 GitHub、Slack、或任何聊天室。
  3. 確認複製好後，按視窗的 CLOSE 關掉。
  4. Credentials 頁面會看到新出現一行 API Key，名字是預設的 API key 1。
萬一忘記複製怎麼辦？不用慌。回 Credentials 頁面，刪掉舊的、再建一個新的就好。建立 Key 沒有次數限制。

拿到 Gemini API Key 了

恭喜，你現在手上有一把可以呼叫 Google Gemini AI 的鑰匙。下面是安全守則，看完再走。

API Key 安全守則

  • 不要 push 到 GitHub。很多新手第一次外洩 Key 都是不小心 commit 進公開 repo。建議把 Key 放在 .env 檔，並把 .env 加進 .gitignore。
  • 不要貼到聊天室或截圖外露。Discord、Slack、LINE 都不行。被別人撿走可能會被偷用 quota，雖然不是直接扣你錢，但你的免費額度會被吃光。
  • 限制 Key 的使用範圍。回 Credentials，點剛建的 Key 進去編輯，把 API restrictions 設成只能用 Generative Language API，這樣就算外洩也只能用這一個服務。
  • 定期換新。長期專案建議每 3-6 個月換一次 Key，舊的刪掉。
  • 萬一外洩怎麼辦？立刻回 Credentials 把那支 Key 刪掉，再建一支新的。舊的一刪就完全失效，不用怕。
如果你的 Key 真的不小心被推到 GitHub，不要只是刪掉那次 commit 就以為沒事。Git 歷史會保留，要立刻去 GCP 把那支 Key 刪掉。爬蟲程式幾分鐘內就會掃到並利用它。

下一步：把 Key 帶回 n8n

有了 Key 之後，回到 n8n 介面，在你要用 AI 的 workflow 節點裡選 Google Gemini，填上這把 Key 就能用了。
